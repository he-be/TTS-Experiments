# スクリプト生成ロジック改修における失敗記録 (2025/12/24)

本ドキュメントは、漫才台本生成プロンプトの改修において発生した問題と、その失敗要因をまとめたものである。

## 取り組みの経緯
ユーザーより以下の課題が指摘されていた。
1.  **ワンパターン**: 生成される台本が単調である。
2.  **論理の飛躍**: B（聞き手）のボケが突飛すぎて、A（ツッコミ）も読者もついていけない。
3.  **禁止事項への抵触**: 「AがBの発言を直接引用してはいけない」というルールが守られていない。

この課題に対し、**「論理の架橋（Logic Bridging）」**と**「直接引用の厳格な禁止」**を主眼としたプロンプト改修を行った。

## 実施した対策（Attempt 2）
プロンプトに以下の指示を追加した。
*   **論理の推論プロセス**: Aが単に否定するのではなく、「なぜBがそのボケに至ったのか（音の類似、字面の類似）」を推論し、解説させる。
*   **引用禁止の強化**: 「Bさんが『〇〇』と言った」という表現をNG例として明示し、要約・換言を求めた。

## 結果：新たな問題の発生
改修後の生成結果（`texts/verification_result_20251224_214502.txt`）において、以下の**「新たなワンパターン」**と**「致命的な欠陥」**が確認された。

### 1. 「解説マシーン化」による単調さ
AがBのボケを逐一、以下の固定テンプレートで解説するようになってしまった。
> 「なるほど、あなたは私の言った『〇〇』という単語の響きに反応し、それを『△△』と聞き間違え、結果として××を連想したのですね？」

これにより、漫才としてのテンポが損なわれ、「ボケ→ツッコミ」ではなく「ボケ→詳細な解説→否定」という、説明臭く退屈な講義のような展開が固定化された。ユーモアが「説明」によって殺されてしまった。

### 2. 入力テーマの無視（指示不履行）
入力されたテーマ（例：「楽器選び」「自販機のジュース」）が無視され、モデルが勝手に「無限」「ブラックホール」といった抽象的・哲学的なテーマを捏造して話し始める現象が発生した。
*   **原因推測**: キャラクター設定の「知的なお嬢様」や、プロンプト内の「論理的に推論せよ」という指示に引きずられ、モデルが「日常的なテーマでは論理推論がしにくい」と判断し、勝手に高尚なテーマにすり替えた可能性がある。

### 3. 不自然な「理解」
「飛躍しすぎて置いてけぼり」を防ぐためにAに理解させた結果、AがBの支離滅裂な思考を一瞬で完璧に読み解くエスパーのようになってしまい、対話としての自然さが失われた。

## 結論
「論理の飛躍を埋めるために、Aに説明させる」というアプローチは、結果として**「説明過多によるマンネリ化」**を招き、失敗であった。
また、プロンプトの制約を強めたことでモデルの柔軟性が失われ、指示されたテーマすら無視する副作用が生じた。
「片側だけ漫才」という制約の中で、自然かつ飛躍しすぎない関係性を作るには、Aに全てを語らせる（説明させる）現在の方針自体を見直す必要があると考えられる。
